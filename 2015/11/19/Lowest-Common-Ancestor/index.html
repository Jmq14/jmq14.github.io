Lowest Common Ancestor of a Binary Tree

问题描述：

Given a binary tree, find the lowest common ancestor (LCA) of two given nodes in the tree.
According to the definition of LCA on Wikipedia: “The lowest common ancestor is defined between two nodes v and w as the lowest node in T that has both v and w as descendants (where we allow a node to be a descendant of itself).”

     _______3______
    /              \
 ___5__          ___1__
/      \        /      \
6      _2       0       8
      /  \
      7   4

For example, the lowest common ancestor (LCA) of nodes 5 and 1 is 3. Another example is LCA of nodes 5 and 4 is 5, since a node can be a descendant of itself according to the LCA definition.

这道题简单来说就是要求我们找到两个节点最近的公共根节点，我们首先要做的就是找到这两个节点，所以会有遍历的过程，那么遍历应该采取哪种遍历方式呢？
我首先是这么考虑的，由于根节点的记录很重要，一旦找到了这两个目标节点，就立即返回最底的共同根节点，这让我想到了中序遍历——先左孩子，再根节点，再右孩子——这样的思路就是把树当成一个地图来搜索，对于根节点的记录，一旦左右子树没有找全两个目标节点，我们所记录的根节点就往上移一层，然后继续到上一层的右子树搜索。
为了不用递归实现这种遍历，我用栈来存储根节点，并同时用一个bool型来标记该节点是否访问过：

stack<pair<TreeNode*, bool>> st;

所以从根节点开始一次遍历，遇到目标节点之后先用指针cr(这大概是某两个单词的首字母缩写，然而我也忘记到底什么意思了= =)指向当前节点，inLine表示已经找到的目标节点数，然后继续遍历，如果回到节点是cr父亲并且还没有找到第二个目标节点，则更新cr为它的父亲节点。
由于严格意义上的中序遍历在访问右孩子之后直接回到根节点的根节点了，所以我添加了一次回根节点的遍历。
遍历的循环如下：

while(!st.empty()){
    m = st.top().first;
    visited = st.top().second;
    st.pop();
    if (!m) continue;
    if ((m == p || m == q)&& !visited){
        inLine++;
        if (inLine == 1) {cr = m;}
        if (inLine == 2) return cr;
    }
    if (visited){
        if (inLine == 1 && (m->left == cr || m->right == cr)) {cr = m;}
    }
    else{
        st.push(make_pair(m, true));
        if (m->right) st.push(make_pair(m->right, false));
        st.push(make_pair(m, true));
        if (m->left) st.push(make_pair(m->left, false));
    }
}

在LeetCode平台上Accepted了，但是运行时间不算快，主要是重复遍历了，应该考虑一下后序遍历，所以还需要优化。
