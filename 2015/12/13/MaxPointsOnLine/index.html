Max Points On A Line

问题描述：

Given n points on a 2D plane, find the maximum number of points that lie on the same straight line.

要求一堆点里在同一条线上的点最多的数量。刚开始想了很久，思考有没有什么比较机智的数学方法可以迅速解决这个问题的，显然我失败了…… 感觉无论怎样是需要用O(n^2)的时间遍历所有两个点之间的关系。由初中学习的平面几何可知，两个点(x1,y1)和(x2,y2)所确定的直线为 (y-y1)/(x-x1)=(y2-y1)/(x2-x1),一开始我觉得要存储一条直线的k和b，但是后来发觉没有这个必要，因为我们可以确定一个点之后，计算所有包含这个点的直线的斜率，只要斜率相等，此时直线就重合了。接下来我们需要做的就是在所有计算出的斜率中找到重复次数最多的，如果用数组来存然后遍历一遍的话肯定是很复杂的，于是不如用哈希表，直接以斜率为索引，value值则是它重复的次数，这样只需要遍历一遍所有可能的斜率值就好了。
另外一定要注意有的点有可能重合，而重复的点已经自然在一条直线上了。我把重复的点的key设置成INT_MIN，把斜率无穷大的点的key设置成INT_MAX。

int maxPoints(vector<Point>& points) {
    hash_map<float, int> slopeTimes;
    int ans = 0;
    for (int i = 0; i < points.size(); i++){
        slopeTimes[INT_MIN]=1;
        for (int j = i+1; j < points.size(); j++){
            if (points[i].x == points[j].x && points[i].y == points[j].y) slopeTimes[INT_MIN]++;
            else if (points[i].x == points[j].x) slopeTimes[INT_MAX]++;
            else {
                float slope = (float)(points[i].y - points[j].y)/(float)(points[i].x - points[j].x);
                slopeTimes[slope]++;
            }
        }
        for (auto p : slopeTimes){
            if (p.first == INT_MIN && p.second > ans) ans = p.second;
            if (p.first > INT_MIN && p.second + slopeTimes[INT_MIN] > ans) ans = p.second + slopeTimes[INT_MIN];
        }
        slopeTimes.clear();
    }
    return ans;
}

然而LeetCode居然告诉我

‘hash_map’ was not declared in this scope

额，那换成map试一下吧，然后runtime是32ms。然后网上查了下，应该是leetcode OJ上没有包含库(它应该不属于C++标准库。。)，不过boost::unordered_map实际上就是实现了计算元素的Hash值，根据Hash值判断元素是否相同，我把map改成unordered_map了之后runtime变为24ms。
网上有很多比较STL中map和hash_map的文章，感觉大致以下几点：

  1. map的实现是红黑树，hash_map是哈希表
  2. 用法上，map的key需要定义operator<，而unordered_map或者hash_map需要定义hash_value函数并且重载operator==。
  3. 效果上，普遍来讲用哈希表查找会更快，不过空间耗费更大，当不需要整体排序的时候应该更多考虑使用哈希表
