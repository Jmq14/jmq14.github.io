WAITING TO BE TRANSCENDED.

2015-12-20 | Leetcode, C/C++

RotatedArray

问题描述：

Q1: Suppose a sorted array is rotated at some pivot unknown to you beforehand. (i.e., 0 1 2 4 5 6 7 might become 4 5 6 7 0 1 2). You are given a target value to search. If found in the array return its index, otherwise return -1. You may assume no duplicate exists in the array.
Q2(Follow up for Q1): What if duplicates are allowed? Would this affect the run-time complexity? How and why? Write a function to determine if a given target is in the array.

Q1主要的思路是二叉搜索，在此基础上加入了对于数组中各段单调性的判断。虽然整体思路比较直观，但我把代码弄得比较冗长，很不美观……
解决了Q1，再看看Q2。

Q2的思路大致相同，也是二叉搜索。Q2用时12ms。后来突发奇想用遍历写了一遍，反而只用了8ms……我认为，一方面是判断单调性的判断次数比较多；另一方面，这271个样例中可能大多数样例的数据规模较小，而遍历等朴素的算法思想反而能更快地解决“最后一公里”的问题，所以遍历反而更快。
做这道题的收获不是在于学到了新的数据结构或算法，而是觉得，理论上复杂度低的算法在特定情况下不一定好用。所以以后在算法的设计和选取中，也需要考虑实际应用情景。正如Linux中的strstr就是用蛮力法实现的，当然了，在基因组比对等应用情形，我们还是得用KMP的。
还有一个多小时，继续跪大作业的GUI去了。。
