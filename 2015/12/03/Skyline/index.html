The Skyline Problem

问题描述：

A city’s skyline is the outer contour of the silhouette formed by all the buildings in that city when viewed from a distance. Now suppose you are given the locations and height of all the buildings as shown on a cityscape photo (Figure A), write a program to output the skyline formed by these buildings collectively (Figure B).

The geometric information of each building is represented by a triplet of integers [Li, Ri, Hi], where Li and Ri are the x coordinates of the left and right edge of the ith building, respectively, and Hi is its height. It is guaranteed that 0 ≤ Li, Ri ≤ INT_MAX, 0 < Hi ≤ INT_MAX, and Ri - Li > 0. You may assume all buildings are perfect rectangles grounded on an absolutely flat surface at height 0.

原题目中还举了例子说明，太长了，要看的话点这里

本来没有什么比较好的想法，看了时宸巨巨的文章才觉得自己真的弱成渣了QAQ资识水平低得不要不要的 Anyway他讲的前缀和真的给我很大启发，我觉得这是一个处理区间问题的很巧妙的思路。因此，我们需要对地图进行这样的转换：

这样一来，我们只需要按照x坐标递增的顺序处理每个矩形开始和结束的状态即可（高度为正是开始，高度为负是结束），而这个状态就是在当前x坐标下还没有结束的矩形的最高高度，而如果当前所有矩形都结束则为0。为了达到这个“最高”的目的，显然是需要使用一些比较好的排序方法或者数据结构。在这里优先队列肯定是首要考虑到的，但是STL中的优先队列是无法修改队列中的元素的，这样的话就不能做到在某个矩形结束的时候删除其对应的高度，所以必须自己写一个可以删除中间项的优先队列。也有大神使用了multiset这种看起来相当高端的数据结构（感到自己资识水平又不够了），好像可以进行优化的排序。
我想的话…由于我们只需要最大值，所以不妨试一下使用最大堆。似乎得解释一下最大堆是什么… 嗯，首先，如果一颗树所有结点的值都大于或等于其叶子结点的值的话，它就叫做最大树，而当它是一个完全二叉树的时候，它就被叫做最大堆了~ 所以一个最大堆的根节点一定是这一坨数据的最大值。
STL中有提供堆的初始化、插入、删除和排序的函数，针对的对象是vector，需要包含库algorithm。在cplusplus references中对这些函数的说明大意是，make_heap是将一个向量变成一个最大或者最小堆，push_heap是在向量已经是一个堆的状态下，将向量尾部加进的新元素放到堆中，而pop_heap同样必须作用在向量已经是一个堆的状态下，然后将first结点与last结点进行交换，然后把[first,last)中的数据再次从上到下地堆化。另外它们的复杂度是O(n)。
值得注意的是，push_heap函数调用之前进行向量的push_back，pop_heap函数调用之后再进行向量的pop_back。官方给出的样例程序如下：

#include <iostream>     // std::cout
#include <algorithm>    // std::make_heap, std::pop_heap, std::push_heap, std::sort_heap
#include <vector>       // std::vector

int main () {
  int myints[] = {10,20,30,5,15};
  std::vector<int> v(myints,myints+5);

  std::make_heap (v.begin(),v.end());
  std::cout << "initial max heap   : " << v.front() << '\n';

  std::pop_heap (v.begin(),v.end()); v.pop_back();
  std::cout << "max heap after pop : " << v.front() << '\n';

  v.push_back(99); std::push_heap (v.begin(),v.end());
  std::cout << "max heap after push: " << v.front() << '\n';

  std::sort_heap (v.begin(),v.end());

  std::cout << "final sorted range :";
  for (unsigned i=0; i<v.size(); i++)
    std::cout << ' ' << v[i];

  std::cout << '\n';

  return 0;
}

Output:
initial max heap   : 30
max heap after pop : 20
max heap after push: 99
final sorted range : 5 10 15 20 99

自己写一个堆的插入和删除也不是太难（感觉至少比排序树好…吧），需要注意的就是这里是用向量的下标来表示这个完全二叉树树的结点关系，所以需要找好对应关系。
由于这道题的存储高度的向量是从empty开始一个一个插入删除，并且每一次都需要取出最大值，所以只需用到push_heap和pop_heap两个函数。

当然这道题还需要注意很多细节，比如和已经记录的前一个节点高度相同的话就不能添加，比如遇到x坐标相同的点的话就得等它们的高度都全部添加或删除之后再做处理等等。感觉这些细节更让人头疼啊。最后修复了各种bug之后终于AC了，但是runtime感人…倒数10%，简直了… 大概还是大神们的选择更明智一些OwO
